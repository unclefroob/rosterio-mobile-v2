## Bug: Post-login navigation shows login/account screen

### Root cause
`AuthContext.js` used `account.id` instead of `account.id || account._id` when
constructing the SIGN_IN payload after auto-selecting an account. The shiftos
backend returns MongoDB documents with `_id`, so `account.id` was `undefined`.
`isAuthenticated = state.isSignedIn && state.selectedAccount` = `true && undefined`
= falsy → RootNavigator rendered AuthNavigator instead of AppTabs.

### Fixed
- `signIn` path 2 (auto-select during login): `account.id` → `account.id || account._id`
- `bootstrapAsync` path (auto-select on token restore): same fix
- Added guard: if `resolvedAccountId` is still falsy, fall through to AccountSelectScreen
