## Bug: Market and Shifts screens display blank/white content

### Root cause
App.js imported `createStackNavigator` from `@react-navigation/stack` which was NOT
installed in node_modules. Only `@react-navigation/native-stack` was available (as a
transitive dep). Metro could not resolve the import, causing MarketplaceNavigator and
MyShiftsNavigator to silently fail at render time. Dashboard worked because it used a
direct Tab.Screen component with no nested Stack wrapper.

Additionally, `@react-navigation/bottom-tabs` was only nested inside expo-router's
node_modules — inaccessible to Metro when importing from the project root.

### Fixed
1. **App.js**: `createStackNavigator` → `createNativeStackNavigator` from `@react-navigation/native-stack`
2. **App.js**: `glassHeaderOptions` updated for native-stack API (`headerShadowVisible: false`)
3. **package.json**: Added `@react-navigation/bottom-tabs` and `@react-navigation/native-stack` as explicit top-level deps
4. **MarketplaceScreen**: null/NaN guards on dates in `renderShiftCard` (secondary crash vector)
5. **MarketplaceScreen**: `setLoading(true)` at top of `loadShifts()` to prevent premature empty state
6. **Both screens**: `navigation.navigate("shift-details")` → `"ShiftDetails"`, `"my-claims"` → `"MyClaims"`
7. **Both screens**: `flexGrow: 1` on FlatList contentContainerStyle
8. **MyShiftsScreen**: removed duplicate `lazy` prop from TabView
